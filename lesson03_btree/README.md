## 核心概念

### 为什么用 B+树？

| 数据结构 | 查找 | 插入 | 范围查询 | 磁盘友好 |
|----------|------|------|----------|----------|
| 哈希表   | O(1) | O(1) | 不支持   | 差       |
| 二叉搜索树 | O(log n) | O(log n) | 支持 | 差（树高大）|
| B+树     | O(log n) | O(log n) | 支持 | 好（节点大，树矮）|

B+树的关键优势：
- **树矮**：每个节点可以存放很多键，百万条记录只需 3-4 层
- **磁盘友好**：一个节点 = 一个页面，一次 I/O 读取整个节点
- **范围查询高效**：叶子节点用链表连接，范围扫描只需遍历链表

### B+树结构

```
阶数 = 3（每个节点最多 2 个键，最少 1 个键）

                    [30]                    ← 内部节点（只存键，不存数据）
                   /    \
              [10,20]   [40,50]             ← 内部节点
             /  |  \    /  |  \
           [5] [15] [25] [35] [45] [55]    ← 叶子节点（存键+数据，用链表连接）
            ↓    ↓    ↓    ↓    ↓    ↓
           data data data data data data
```

### 内部节点 vs 叶子节点

```
内部节点（InternalNode）：
  ┌────┬────┬────┬────┬────┐
  │ P0 │ K1 │ P1 │ K2 │ P2 │
  └────┴────┴────┴────┴────┘
  P0 指向所有键 < K1 的子树
  P1 指向 K1 ≤ 键 < K2 的子树
  P2 指向所有键 ≥ K2 的子树

叶子节点（LeafNode）：
  ┌────┬────┬────┬────┬──────┐
  │ K1 │ V1 │ K2 │ V2 │ next │
  └────┴────┴────┴────┴──────┘
  存储实际的键值对，next 指向下一个叶子节点（范围查询用）
```

### 节点分裂（Insert 时）

```
插入 25 到已满节点 [10, 20, 30]（阶数=3，最多2个键）：

分裂前：[10, 20, 30]（溢出）
分裂后：[10] 和 [25, 30]，中间键 20 上推到父节点
```

## 时间复杂度分析

- 树高 h = O(log_m n)，m 是阶数（节点最大键数）
- 查找：O(log n)
- 插入：O(log n)（可能触发 O(log n) 次分裂）
- 删除：O(log n)（可能触发 O(log n) 次合并）
- 范围查询 [a, b]：O(log n + k)，k 是结果数量
